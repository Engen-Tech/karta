# Multi-binder sequences Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let `karta-plan` emit an ordered set of self-sufficient binders when a job separates into stages that must land in order, advise the run order once at plan time, and persist nothing about the order.

**Architecture:** A `karta-plan`-only change. Three edits to `skills/karta-plan/SKILL.md` (set emission in Phase 5, order advice in Phase 6, drop the one-binder-per-run language), one note in `skills/karta-plan/references/binder-reference.md`, and a worked expand→migrate→contract example under `skills/karta-plan/references/example-sequence/`. No schema change, no validator change, no new runtime artifact. The Codex mirrors regenerate from the canonical edits.

**Tech Stack:** Markdown SKILL/reference files (Claude + Codex plugin), JSON binder examples, the existing pure-stdlib `validate_binder.py` (unchanged), `uv`-run maintenance scripts.

## Global Constraints

Every task's requirements implicitly include these, copied from the spec (`docs/specs/2026-06-22-multi-binder-sequences-design.md`) and repo conventions:

- **Default stays one binder.** A sequence is the exception, not the reflex — never split work that fits one binder.
- **Each binder is self-sufficient and independently mergeable**, sliced expand → migrate → contract so each leaves the default branch green on its own. Each must pass `validate_binder.py` standalone.
- **Order is derived advice, never stored.** NO manifest file, NO order baked into slugs, NO `after` field, NO new file under `.karta/`.
- **No binder-schema change. No `validate_binder.py` change.**
- **Slugs are descriptive and unique, grouped by a shared prefix, and carry no sequence number** (a number is a stored order that rots).
- **Validate-each, commit-together**: every binder in a set validates before the set is presented; the whole set commits on the explicit `commit` verb.
- **Canonical vs generated**: `skills/<name>/` is canonical; `.agents/`, `plugins/karta/` are GENERATED — never hand-edit. After editing any skill, run `uv run scripts/sync_codex_skills.py`.
- **Four pre-commit checks must be green** after every task:
  - `uv run scripts/validate_plugin.py --self-test`
  - `uv run scripts/check_shared_copies.py --self-test`
  - `uv run scripts/sync_codex_agents.py --check`
  - `uv run scripts/sync_codex_skills.py --check`
- **Validator invocation form is `uv run --script <path>`.**
- **Markdown tables use minimal separators (`|-|-|`); never box-drawing characters.**
- **Commit messages end with:**
  ```
  Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
  Claude-Session: https://claude.ai/code/session_012govqoYUPMxPTgLXw2Qic7
  ```
