# karta 2.21.0 — release test-drive checklist

Karta 2.21.0 adds one thing: the **delivery Stop-gate** — a sixth Claude Code plugin hook that keeps a session from quietly ending while a delivery is dirty. Two states trigger it, both read from git alone: an item built but never merged (a `built` ref with no `done` and no `failed`), and a binder whose items are all done but whose archive is committed nowhere that counts. The gate blocks the stop once per session per state with the exact fix in the reason, then lets the next identical stop through — a nudge, not a wall. It is fail-open: an unexpected internal error always allows.

Design: [2026-07-14-delivery-stop-gate-design.md](../specs/2026-07-14-delivery-stop-gate-design.md) — Stop-hook capability verified against the installed binary (v2.1.207), 4-model adversarial critique folded in before build.

## Test-drive (run with the 2.21.0 plugin installed)

### 1. Built-unmerged nudge

- [ ] In a scratch repo with a live binder, hand-write a `refs/karta/<slug>/item-<id>/built` ref with no `done`, then end a turn; confirm the stop is blocked once with the slug and item named and the merge-queue fix quoted, and that the very next stop with the same state passes (sentinel recorded).
- [ ] Add a `failed` ref beside the `built` ref; confirm the gate goes silent (a parked four-way-choice state is not a finding).

### 2. Complete-unarchived nudge

- [ ] Give every item a `done` ref with the binder still live and no archive commit anywhere; confirm one block naming the end-of-life step.
- [ ] Commit the archive on the binder's integration branch only; confirm the gate allows (awaiting-human-merge is the correct end state).

### 3. Non-interference

- [ ] A repo with only archived binders (surviving refs included) never triggers the gate.
- [ ] `stop_hook_active: true` in the payload always allows (loop guard).
- [ ] A changed dirty state (one more item merged, one still stranded) blocks once more; an unchanged state never re-fires in the same session.
