# karta 1.17.0 — release test-drive checklist

A focused, incremental checklist. 1.17.0 adds one thing: **binder end-of-life** — a delivered binder is archived, its slug retired, so finished work stops haunting status. The [1.16.0 checklist](v1.16.0-checklist.md) still certifies the plannotator review surface, [1.15.0](v1.15.0-checklist.md) the evidence-and-dogfood layer, and [1.14.0](v1.14.0-checklist.md) the enforcement chain.

## What 1.17.0 adds — and what this checklist is for

- **Delivered binders are archived, not left to rot.** On a complete run, karta-deliver's new Phase 6c (`deliver:archive`) moves the binder to `.karta/binders/archive/<slug>.json` as a commit on the integration branch — the archival merges to the default branch together with the delivered work. Incomplete runs (deferred or halted items) leave the binder live; the step is resume-idempotent. In the single-item hatch, karta-build's 9c-single owns the same step (no orchestrator exists there).
- **Status only shows live work.** `karta-status` and the session-start summary list only `.karta/binders/*.json`; the watch page keeps history by rendering archived binders under its Delivered phase. An `after` edge naming an archived binder resolves as satisfied — a delivered predecessor is never a dangling warning (engine and `validate_binder.py` both).
- **Slugs are retired, never recycled.** The delivered run's `refs/karta/<slug>/` refs survive archival, so a namesake binder would read the old `done` refs as its own and silently skip work. karta-plan's slug rule requires a fresh slug; the validator and status engine warn when a live binder shadows an archived one.
- **Delivered history is read-only.** The committed-binder immutability hook now also covers `.karta/binders/archive/`; re-running a delivered slug gets "already delivered — archived at …" from the validator instead of a traceback.

## Test-drive (run 2026-07-10, on karta 1.19.0)

Certified retroactively — two releases late, with the feature already in daily use. Where a box's scenario had since happened for real, the real occurrence is the evidence; the rest ran as fixtures. Section 1's complete-delivery box is covered by the two 2026-07-09 deliveries (`writer-confinement`, `shared-terms-pending`: archive commits on their integration branches, terminal map silent after merge, watch page rendering both as merged rows under Delivered — observed live via `/state.json`). The deferred case ran as a git fixture (one `done` ref, one `failed`; the binder stayed live and a later status run pointed at the remaining work). The single-item hatch ran for real in a fixture repo: a one-item binder built through the full 9c-single sequence, ending in the worker's own `chore(karta): archive binder greet — delivered` commit and the end-of-life footer. Section 2 ran as validator/status fixtures against the real repo's archive; section 3 was a live in-session denial — an `Edit` on `.karta/binders/archive/writer-confinement.json` blocked by the plugin hook with the archive rule named.
